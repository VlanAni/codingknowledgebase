# ArrayList в Java: Внутреннее устройство

> [!info] Базовые принципы
> **ArrayList** — это динамический массив, реализующий интерфейс `List`. В отличие от классических массивов в Java, которые имеют строго фиксированный размер, `ArrayList` может автоматически расширяться.
> - Элементы могут быть любых типов, включая `null`.
> - Не синхронизирован (не потокобезопасен).

## 1. Внутренние свойства (Создание объекта)
При вызове `new ArrayList<String>()` под капотом создается объект со следующими свойствами:
- `elementData` — обычный массив `Object[]`, который является физическим хранилищем наших элементов. Если емкость не указана, по умолчанию создается массив на **10 элементов**.
- `size` — счетчик, указывающий на фактическое количество элементов, добавленных в список (а не на размер массива `elementData`).

## 2. Добавление элементов

### Добавление в конец (`add(value)`)
1. Вызывается метод `ensureCapacity(size + 1)`, который проверяет, осталось ли в массиве `elementData` свободное место.
2. Если места достаточно, элемент просто записывается в следующую свободную ячейку: `elementData[size++] = element;`.
3. **Если массив заполнен:**
   - Вычисляется новая емкость (capacity) по формуле: `(oldCapacity * 3) / 2 + 1` (то есть массив увеличивается примерно в 1.5 раза).
   - Создается новый массив большего размера.
   - С помощью быстрого нативного метода `System.arraycopy()` все данные из старого массива копируются в новый.

### Добавление в середину (`add(index, value)`)
1. Проверка вместимости `ensureCapacity()` (с возможным копированием массива, если места нет).
2. С помощью `System.arraycopy()` элементы, начиная с позиции `index`, сдвигаются на одну позицию вправо, освобождая ячейку.
3. В освободившуюся ячейку `elementData[index]` записывается новый элемент.
> [!warning] Проблема производительности
> Если при вставке в середину массив оказался полностью заполнен, `System.arraycopy()` отработает **дважды**: первый раз для расширения массива, второй — для сдвига элементов. Это делает операцию добавления в середину списка медленной.

## 3. Удаление элементов

Удаление может происходить по индексу `remove(index)` или по значению `remove(value)`. 
Процесс удаления по индексу:
1. Вычисляется количество элементов, которые необходимо сдвинуть влево: `size - index - 1`.
2. Происходит сдвиг всех правых элементов влево на одну позицию через `System.arraycopy()`.
3. В последнюю ячейку (которая теперь дублируется) явно записывается `null` (`elementData[--size] = null;`), чтобы Garbage Collector (Сборщик мусора) мог освободить память.
При удалении по значению список просматривается линейно, пока не найдется первое совпадение, после чего выполняется удаление по его индексу.

> [!tip] Утечки памяти (Memory Leaks)
> При удалении элементов физический размер массива `elementData` (capacity) **не уменьшается**. Если вы удалили много объектов, резерв памяти останется. Чтобы освободить память вручную, нужно вызвать метод `trimToSize()`, который обрежет массив до фактического значения `size`.

## 4. Итоговая вычислительная сложность
- **Доступ по индексу (`get`)**: $O(1)$ — очень быстро.
- **Доступ по значению (`indexOf`, `contains`)**: $O(n)$ — требует линейного перебора.
- **Вставка / Удаление в конце**: $O(1)$ (амортизированное, так как изредка требуется время $O(n)$ на расширение массива).
- **Вставка / Удаление в середине**: $O(n)$ — очень медленно из-за необходимости сдвигать все последующие элементы массива.
