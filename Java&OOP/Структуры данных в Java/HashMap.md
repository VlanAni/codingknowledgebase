# HashMap в Java: Внутреннее устройство

> [!info] Базовые принципы
> **HashMap** — структура данных, основанная на хэш-таблицах, реализующая интерфейс `Map`. Хранит данные в виде пар «ключ-значение». 
> - Разрешает использование любых типов ключей и значений, включая `null`.
> - **Не гарантирует** сохранение порядка элементов.
> - Коллизии разрешаются **методом цепочек** (связные списки внутри ячеек массива).
> - Не синхронизирован (не потокобезопасен).

## 1. Внутренние свойства (Создание объекта)
При создании `new HashMap<String, String>()` инициализируется объект со следующими ключевыми свойствами:
- `table` — массив типа `Entry[]`, который выступает хранилищем ссылок на списки (цепочки) элементов. Базовая емкость (capacity) по умолчанию равна **16**.
- `loadFactor` — коэффициент загрузки. По умолчанию равен **0.75** (оптимальный баланс между временем доступа и затратами памяти).
- `threshold` — предельное количество элементов, при котором массив увеличится вдвое. Рассчитывается как `capacity * loadFactor` (по умолчанию $16 \times 0.75 = 12$).
- `size` — текущее количество элементов в коллекции.

## 2. Процесс добавления элемента (`put(key, value)`)
Добавление происходит в несколько строгих этапов:

1. **Проверка на `null`**: 
   Если `key == null`, вызывается отдельный метод `putForNullKey(value)`. Элементы с null-ключами *всегда* помещаются в ячейку `table[0]`.
2. **Генерация хэша**: 
   Берется стандартный `key.hashCode()` и прогоняется через внутренний метод `hash(hashCode)`, который выполняет дополнительные побитовые сдвиги. Это нужно, чтобы минимизировать коллизии.
3. **Определение индекса в массиве**: 
   С помощью функции `indexFor(hash, tableLength)` вычисляется позиция. Используется побитовая операция: `h & (length - 1)`.
4. **Поиск совпадений и запись**:
   По вычисленному индексу программа обращается к ячейке массива `table` и проходит по всей цепочке элементов:
   - Если находится элемент с таким же хэшем и ключом, его значение (`value`) перезаписывается новым, а старое значение возвращается.
   - Если совпадений нет (или ячейка пуста), вызывается метод `addEntry()`. Новый элемент вставляется **в начало цепочки** (становится первым в списке, а его ссылка указывает на предыдущий первый элемент).

## 3. Перераспределение памяти (Resize и Transfer)
Когда количество элементов `size` достигает предела `threshold`:
- Вызывается метод `resize(capacity)`, который создает новый массив `Entry[]`, размер которого **в 2 раза больше** предыдущего.
- Запускается метод `transfer(newTable)`: он проходит по всем элементам старой таблицы, **заново пересчитывает их индексы** (так как `length` массива изменилась) и распределяет их по новым ячейкам.

## 4. Удаление элементов
> [!warning] Особенность работы с памятью
> При удалении элементов из HashMap размер базового массива `table[]` **не уменьшается**. Механизма очистки пустых ячеек под капотом нет (в отличие от `ArrayList`, у которого есть `trimToSize()`). Если вы удалили огромное количество элементов, объект всё равно будет занимать много памяти. Для обхода этой проблемы можно пересоздать коллекцию: `hashmap = new HashMap<K, V>(hashmap);`

## 5. Итераторы и перебор
Для перебора значений HashMap предоставляет методы: `keySet()` (ключи), `values()` (значения) и `entrySet()` (пары ключ-значение).
*Важно:* Если структура HashMap была изменена во время работы итератора извне (не методами самого итератора), поведение становится непредсказуемым.

## 6. Итоговая вычислительная сложность
- **Вставка**: $O(1)$ (новый элемент просто добавляется в начало связного списка).
- **Поиск / Удаление**: в среднем $O(1)$ (если хэш-функция хорошая и коллизий мало). В худшем случае (когда все элементы попали в одну корзину из-за коллизий) — $O(n)$.
