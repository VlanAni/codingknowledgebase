Для просмотра списка удалённых репозиториев используется команда `git remote`: 
- `git remote` без аргументов выводит обычный список **привязанных удалённых репозиториев** 
- `git remote -v` выводит помимо имён также **адреса** удалённых репозиториев 
### Добавление ссылки на удалённый репозиторий 

Добавление удалённого репозитория: `git remote add <short_name> <URL>` 

`short_name` - короткое имя для репозитория 
### Получение изменений из удалённого репозитория — Fetch и Pull

`git fetch [remote-name]` - получение данных, которые не имеются на локальном репозитории, из удалённого репозитория с именем `remote-name` 

> [!info] Важный момент
>  При выполнении `git clone` автоматически добавляется клонируемый репозиторий с именем `origin` 

`git pull` - это команда, которая не только **подгружает** изменения из удалённого репозитория, но и **мержит** эти изменения с локальными ветками, которые настроены на отслеживание соответствующих **удалённых** веток 

> [!info] Важный момент 
> При выполнении `git clone` автоматически настраивается отслеживание главной локальной веткой главной удалённой ветки 
### Отправка изменений в удалённый репозиторий (Push)

Команда `git push [remote-rep] [branch-name]` позволяет загрузить ветку `branch-name` в удалённый репозиторий `remote-rep` 

> [!warning] Права на запись 
> `push` сработает только в том случае, если у вас есть **право на запись**
### Просмотр информации об удалённом репозитории 

Для просмотра информации об удалённом репозитории с именем `remote-name` нужно использовать команду `git remote show [remote-name]`
